# 口コミデータフォルダ

このフォルダには、口コミ自動投稿用のテキストファイルを配置します。

## ファイル形式

### 1. カテゴリ別ファイル（推奨）

以下のファイル名でカテゴリごとに口コミを分けて管理できます：

- `reviews-chuo.txt` - 中央競馬（JRA）サイト用の口コミ
- `reviews-nankan.txt` - 南関競馬サイト用の口コミ
- `reviews-chihou.txt` - 地方競馬サイト用の口コミ

### 2. 評価別ファイル（推奨）

サイトの品質評価ごとに口コミを分けることもできます：

- `reviews-malicious.txt` - 悪質サイト用（1-2星）
- `reviews-normal.txt` - 通常サイト用（3星）
- `reviews-excellent.txt` - 優良サイト用（4-5星）

### 3. ファイルフォーマット

各ファイルは以下の形式で記述してください：

```
タイトル1
口コミ本文1

タイトル2
口コミ本文2

タイトル3
口コミ本文3
```

**重要な注意点：**
- タイトルと本文の間は改行なし
- 各口コミの間は**空行1行**で区切る
- 本文は80〜150文字が推奨（SEO最適化）
- タイトルは簡潔に（10〜20文字）

## 例

```
期待外れでした
3ヶ月利用しましたが、有料情報の的中率が10%程度でした。情報料が1回3万円と高額なのに、サポートの返信も遅く不満です。費用対効果を考えると全くおすすめできません。

まあまあ良い
無料予想を2週間試してみましたが、的中率は30%くらいで普通です。予想の根拠が書いてあるのは良いですが、有料プランは少し高いと感じました。

的中率が高め
初心者にも分かりやすい予想で、無料情報でもそこそこ当たります。サポート対応も丁寧で安心して利用できました。有料情報は少し高めですが、それに見合う価値はあると思います。
```

## 使用方法

1. 口コミテキストファイルをこのフォルダに配置
2. スクリプトで読み込んで自動投稿

```bash
# カスタム口コミを使った投稿（将来実装予定）
node scripts/post-custom-reviews.cjs
```
